# Support

Syx is an experimental research prototype maintained on a best-effort basis. It is not a hosted product, production service, or supported commercial offering.

There is no guaranteed response time for questions, issues, pull requests, setup problems, feature requests, or security reports.

## Where to Ask

### General questions and design discussion

Use GitHub Discussions for:

- Questions about Syx concepts.
- Ideas about local AI memory, RAG, Sleep, Dream, or agent memory.
- Design discussions.
- Experiment notes.
- Usage questions that are not clearly bugs.

### Bugs

Use GitHub Issues for reproducible bugs.

A useful bug report should include:

- What you were trying to do.
- What happened.
- What you expected to happen.
- Steps to reproduce.
- Relevant logs or error messages.
- Operating system.
- Python version.
- Node version.
- Whether you were running locally or with Docker.

Please do not include API keys, `.env` files, private memory artifacts, uploaded documents, chat logs, personal data, or generated indexes in bug reports.

### Feature requests

Use GitHub Issues or Discussions for feature ideas.

Feature requests should describe the use case, not only the desired implementation.

Large features, architecture changes, new providers, storage engine changes, hosted deployment support, production authentication, MCP changes, or agent write-back workflows may not be accepted without prior discussion.

## What Is Not Promised

This project does not currently promise:

- Setup support.
- Debugging support for local environments.
- Consulting or implementation help.
- Guaranteed feature work.
- Pull request review timelines.
- Compatibility with all operating systems.
- Production deployment support.
- Public internet deployment support.

## Security

Syx is intended for local or trusted environments only. Do not expose the API directly to the public internet or to untrusted users.

For security concerns, see `SECURITY.md`.

## Maintainer Availability

Maintainer availability is limited. Issues, Discussions, and pull requests may be delayed or closed if they are too broad, inactive, outside project scope, or not reproducible.
